HTML Basics

These flashcards drill on HTML basics with code snippet examples where applicable. The main sources of reference material for these cards are w3schools.com and Mozilla developer documentation


HTML
Hypertext Markup Language is a markup language rather than a programming language, which means it defines how a page will be presented rather than how elements behave. It holds all the instructions for how the web page is displayed by ordering elements that enclose the content.
CSS
Cascading Style Sheets is a language that controls the styling of the page and how it appears. It can be added or linked in the HTML style tag, and its benefit is its flexibility. CSS isn't necessary to build a webpage, but utilizing it allows design elements to be clumped and repeated throughout the layout with no inline, repetitive style attributes.
element
Elements are the individual building blocks of HTML. Most have a beginning tag (<p>) and an ending tag(</p>). A few tags like <img> and <br> are called empty tags and only have a starting tag.
Elements of a Basic HTML doc

tags
One half of the element. An opening tag has a beginning angle bracket(<), the content (i.e. p or div), and an ending angle bracket with a forward slash(/>).
    Type Examples:
  • <p> - paragraph; always starts on a new line
  • <div> - general divider/grouping of elements. Most widely used type of element
  • <h1> - heading tags make larger text by default than div or p. There are h1 through h6 with h1 being the largest. Search engines use them to structure and index content, so headings should not be used for size or boldness, only for structuring the document.
  • <a> - anchor tags hold links
  • <img> - image tags hold images
  • <span> - span tags hold content so CSS or JS can easily find and be applied to it.

links
A link can be any HTML element like a button though it is usually text or an image. E.g. <a href="default.asp"><img src="smiley.gif" alt="HTML tutorial" style="width:42px;height:42px;"></a>
attributes
Give extra information about the element they live in. All elements can have one or more attributes. They always appear in the starting tag of the element they belong to. In <a href="google.com">Google link</a>, <a> is the anchor element to hold a link, href="" is the attribute that holds the destination URL, and Google link is the inner HTML that will be the hyperlink.
Attributes are usually structured like name_of_attribute = "value_of_attribute".
<img> element attributes: src, width/height, alt, and style
    src=""
  1. Absolute URL - goes straight to specified website, but beware this option for copyright infringement or if the link is removed at some point in the future
  2. Relative URL - the file/image is saved locally and goes to specified location on your machine/where it is hosted. Better option.
    width="" and height=""
  1. If these are not set as attributes, the webpage will size strangely as it is loading up. It helps reserve the space that will be needed.
    alt=""
  1. accessibility feature that should always be used for images
  2. If the image cannot be displayed, the text will appear instead, and it will also help screen readers describe to visually impaired users what the image contains
    style=""
  1. More often styling is done with CSS instead
  2. i.e. style="color: red"

accessibility tips with attributes: alt, title
  1. <img alt=""> - If the image cannot be displayed, the text will appear instead, and it will also help screen readers describe to visually impaired users what the image contains. Mostly seen with the img element.
  2. <p title=""> - this attribute makes extra information appear whenever the user hovers over this element. Can be used with most elements.

  • <hr> - empty tag that acts like <br> but shows a horizontal line across the page.
  • <pre> - preformatted text holds the format of the spacing in its inner HTML. Good for poetry styles.

HTML & Color: RGB & RGBA
  • RGB (red, green, blue) - each value is from least to most intense (0-255)
      Examples of pure colors:
    1. RED = rgb(255, 0, 0)
    2. GREEN = rgb(0, 255, 0)
    3. BLUE = rgb(0, 0, 255)
    4. BLACK = rgb(0, 0, 0)
    5. WHITE = rgb(255, 255, 255)
    6. GREY (all values equal) rgb(20, 20, 20) or rgb(200, 200, 200)
  • RGBA (red, green, blue, alpha) - rgb are still 0-255, and alpha (opacity) is 0.0(fully transparent) to 1.0 (fully opaque)
    Syntax for all color use:
  • CSS - body {background-color:rgb(255, 95, 71)}
  • Inline HTML - <h1 style="background-color:rgb(255, 95, 71)">
HTML & Color: HEX
    HEX (hexidecimal: rrggbb) - each digit is from least to most intense (0-9, then a-f)
      Examples of pure colors:
    1. RED = #ff0000
    2. GREEN = #00ff00
    3. BLUE = #0000ff
    4. BLACK = #000000
    5. WHITE = #ffffff
    6. GREY = (all values equal) #404040 or #dcdcdc
HTML & Color: HSL & HSLA
  • HSL (hue, saturation, light) - hue is a degree on the color wheel (0-360), saturation is a % value (0% is grey - 100% is full color), and light is a % value (0% is black - 100% is white)
      Examples of pure colors:
    1. RED = hsl(0, 100%, 50%)
    2. GREEN = hsl(147, 50%, 50%)
    3. BLUE = hsl(240, 100%, 50%)
    4. BLACK = hsl(0, 100%, 0%)
    5. WHITE = hsl(0, 100%, 100%)
    6. GREY = hsl(any number, 0%, 100%) - any color can have its grey equivalent
  • HSLA (hsl with alpha as an opacity value). 0.0 is fully transparent to 1.0 is fully opaque.
